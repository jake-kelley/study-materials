Full guide

Foundations

  • CIA + Non-repudiation. AAA = Authn / Authz / Accounting.
  • Controls category: Technical/Managerial/Operational/Physical.
  • Control type: Preventive/Deterrent/Detective/Corrective/Compensating/Directive.
  • Zero Trust: Control Plane (policy engine/admin, adaptive identity) + Data Plane (PEP, subject).

Cryptography

  • Symmetric (1 key, fast, bulk): AES, ChaCha20, 3DES.
  • Asymmetric (key pair, exchange+signatures): RSA, ECC, DH.
  • Hash (integrity, 1-way): SHA-2/256, HMAC. MD5=broken.
  • Digital sig = hash + sender private key → integrity+auth+non-repudiation. Confidentiality = recipient public key.
  • Passwords: salt + bcrypt/PBKDF2/Argon2.

PKI

  • CA, RA, CSR, X.509. CRL (list) vs OCSP (real-time status).
  • Root of trust, wildcard/SAN, key escrow.
  • TLS = hybrid: asym exchanges a sym session key.
  • TPM (on-device) vs HSM (dedicated hardware).

Attacks

  • Social: phish/spear/whale/vish/smish, watering hole, typosquat, BEC.
  • Network: DDoS(amp/reflect), on-path/MITM, DNS/ARP poison, replay, pass-the-hash.
  • App: SQLi, XSS(stored/reflected/DOM), CSRF, buffer overflow, TOCTOU race, dir traversal, priv-esc.
  • Pwd: spray vs brute vs dictionary. Crypto: birthday/collision/downgrade.

IAM

  • MFA factors: know / have / are / somewhere-you-are / something-you-do.
  • SSO: SAML(web SSO), OAuth(authz), OIDC(authn), Kerberos(TGT, 88).
  • RADIUS vs TACACS+ (separates AAA, TCP, Cisco).
  • Models: DAC(owner) MAC(labels) RBAC(role) ABAC(attributes) rule-based.
  • PAM: JIT, vaulting, ephemeral. Least priv, sep of duties.

Security Ops Tools

  • SIEM aggregate/correlate logs; SOAR automate/playbooks.
  • DLP data loss; EDR/XDR endpoint; NAC; 802.1X.
  • Email auth: SPF / DKIM / DMARC.
  • Vuln mgmt: scan, CVE (id), CVSS (score), false pos/neg, patch/mitigate/validate.
  • Firewalls: WAF(L7 web), NGFW, UTM. IDS(detect) vs IPS(block).

Incident Response

Prep → Detect → Analysis → Containment → Eradication → Recovery → Lessons Learned

  • Forensics: chain of custody, legal hold, order of volatility, e-discovery.
  • Data: firewall/app/endpoint/OS/IDS/network logs, pcap, metadata.

Risk & BC

  • SLE = AV × EF; ALE = SLE × ARO.
  • BIA: RTO(restore time) RPO(data loss) MTBF(between failures) MTTR(repair).
  • Strategies: Accept / Avoid / Transfer / Mitigate.
  • Risk register, appetite/tolerance, KRIs.

Frameworks

  • NIST CSF: Identify, Protect, Detect, Respond, Recover (+Govern in 2.0).
  • NIST RMF; 800-53 (fed), 800-171 (CUI).
  • CIS Controls (18)/Benchmarks; ISO 27001/2; SOC 2; PCI DSS.
  • MITRE ATT&CK, Cyber Kill Chain, Diamond, OWASP.
  • 3rd-party agreements: SLA/MOU/MOA/MSA/SOW/NDA/BPA.

Architecture & Data

  • Data states: at rest / in transit / in use. Protect: encrypt, mask, tokenize, obfuscate.
  • fail-open vs fail-closed; inline vs tap; jump server, proxy.
  • Resilience: hot/warm/cold sites, LB vs clustering, geo-dispersion; UPS/generator.
  • Wireless: WPA3 + 802.1X/RADIUS/EAP.
