Well-Architected Pillars

  • Op Excellence · Security · Reliability · Performance · Cost · Sustainability
  • Shared Responsibility: AWS = security OF cloud; you = security IN cloud (data, IAM, OS patching, config).

IAM

  • Global, free. Explicit Deny > any Allow; default = implicit deny.
  • Roles > access keys. EC2 → attach instance role, never hardcode keys.
  • Cross-account = role + sts:AssumeRole, not new users.
  • Policy types: identity, resource (has Principal), permission boundary, SCP.
  • STS = temp creds. Identity Center = SSO. Cognito = customer app identity (User Pool = sign-in, Identity Pool = AWS creds).
  • MFA on root; lock root away; least privilege.

Multi-Account

  • Organizations = many accts + consolidated billing + OUs.
  • SCP = guardrail / max permission ceiling — does NOT grant.
  • Control Tower = automated governed landing zone.
  • RAM = share resources across accounts.

Encryption / Data

  • KMS = managed keys, envelope encryption, auto annual rotation (CMK). CloudHSM = dedicated/single-tenant/FIPS 140-2 L3.
  • ACM = free auto-renew TLS certs (CloudFront needs us-east-1).
  • Secrets Manager = secrets + auto rotation (RDS). Parameter Store = cheaper, no rotation.
  • Macie = find PII in S3. Config = config/compliance history. Artifact = compliance reports.

VPC / Networking

  • SG = instance, stateful, allow-only. NACL = subnet, stateless, allow+deny, numbered order.
  • Public subnet = route 0.0.0.0/0 → IGW. Private outbound → NAT GW (in public subnet, per-AZ for HA).
  • Gateway endpoint = S3/DynamoDB, free, route table. Interface endpoint = PrivateLink, ENI, $.
  • VPC Peering non-transitive. Transit GW = hub for many VPCs+on-prem.

Edge & DNS

  • CloudFront = CDN, cache content at edge, OAC to S3, signed URLs.
  • Global Accelerator = static anycast IPs, route TCP/UDP to nearest healthy region, fast failover.
  • Route 53 policies: Simple, Weighted, Latency, Failover, Geolocation, Geoproximity, Multivalue, IP. Alias = free to AWS resources.

S3

  • 11 nines durability, ≥3 AZ (not One Zone-IA). Object ≤5 TB, multipart >5 GB.
  • 3,500 PUT / 5,500 GET per prefix/s. Transfer Acceleration = edge uploads.
  • Block Public Access on by default; Versioning; Object Lock = WORM.

Block / File Storage

  • EBS = 1 EC2, single-AZ, snapshot→S3. gp3 (tune IOPS/throughput), io2 (high IOPS, Multi-Attach), st1 (throughput HDD), sc1 (cold).
  • Instance store = ephemeral, lost on stop.
  • EFS = NFS, Linux, multi-AZ, elastic. FSx: Windows(SMB/AD), Lustre(HPC+S3), NetApp, OpenZFS.

Compute + Purchasing

  • Families: M/T general, C compute, R/X memory, I/D storage, P/G GPU.
  • Placement: Cluster(low latency/1AZ), Spread(≤7/AZ distinct HW), Partition(HDFS/Cassandra).
  • λ: 15-min max, 128MB–10GB mem (=CPU). Fargate = serverless containers.

Databases

  • RDS Multi-AZ = HA (sync standby, failover) — NOT read scaling. Read replica = async read scaling (cross-region OK).
  • Aurora = 6 copies/3 AZ, 15 replicas, auto-heal, Serverless v2.
  • DynamoDB = serverless KV, single-digit ms, on-demand/provisioned, Global Tables, Streams. DAX = microsecond cache.
  • ElastiCache Redis/Memcached = cache/sessions. Redshift = OLAP warehouse. RDS Proxy = pool conns (λ).
"Improve reads" → replica/cache. "Survive AZ" → Multi-AZ.

Scaling / Serverless

  • ASG across AZs; policies: target-tracking, step, scheduled, predictive. + ELB health checks.
  • Make instances stateless → session in DynamoDB/ElastiCache.
  • Containers: ECS(AWS-native), EKS(K8s), on EC2 or Fargate. ECR=registry.

Data / Analytics

  • Kinesis Data Streams = real-time, shards, custom consumers, replay. Firehose = no-code delivery to S3/Redshift/OpenSearch.
  • Glue = serverless ETL + catalog (CSV→Parquet). Athena = SQL on S3 (pay/scan). EMR = Spark/Hadoop. QuickSight = BI.
  • Move data: DataSync(online), Snow(offline PB), Storage Gateway(hybrid).

Cost Tools + Network Cost

  • Cost Explorer analyze, Budgets alert, CUR granular→S3, Trusted Advisor checks, cost allocation tags.
  • Same-AZ private IP = free; cross-AZ/region/internet = $. Gateway endpoints avoid NAT + egress. CloudFront cuts origin egress.

Keyword → Answer

  • "decouple/buffer" → SQS
  • "notify many/fan-out" → SNS
  • "least ops / managed / no servers" → serverless (λ, Fargate, Aurora Serverless, S3)
  • "highly available" → Multi-AZ / ASG multi-AZ
  • "durable / archive cheap" → S3 / Glacier
  • "move PBs / no bandwidth" → Snow
  • "cross-account guardrail" → SCP
  • "temporary creds" → role / STS
  • "unknown access pattern" → S3 Intelligent-Tiering
  • "microsecond DynamoDB" → DAX

Top Traps

Multi-AZ ≠ read scaling · SCP ≠ grant · EBS single-AZ · Spot ≠ stateful DB · CloudFront(cache) vs Global Accelerator(route) · Gateway(free) vs Interface(paid) endpoint · KMS vs CloudHSM(dedicated/FIPS-L3) · Kinesis Streams(custom) vs Firehose(no-code)
Read the last sentence first: optimize for cost / least-ops / HA / latency / secure — that word picks between two "right" answers.
