AZ-305: Designing Microsoft Azure Infrastructure Solutions

Passing AZ-305 (plus holding AZ-104) earns the Azure Solutions Architect Expert certification. The exam is design-oriented: given business/technical requirements, choose the best Azure service and configuration — across identity/governance, storage, business continuity, and infrastructure. Heavy on case studies.

~40–60 questions ~100–120 min pass 700/1000 $165 USD AZ-104 prereq (for cert)
⚡ Open the 1-page Cram Jam sheet

Part 1 — High-Level Overview

What & Why

AZ-305 validates your ability to design solutions on Azure that meet requirements — it's the architect-level companion to the administrator-level AZ-104. You won't be clicking through the portal; you'll be reading a scenario and picking the right service, tier, and topology. The mental model is Microsoft's Well-Architected Framework (WAF) and Cloud Adoption Framework (CAF).

Cert requirement The Azure Solutions Architect Expert certification requires both AZ-104 (Administrator) and passing AZ-305. You can sit AZ-305 first, but the Expert cert isn't awarded until AZ-104 is earned — and AZ-104's knowledge is assumed. Detail in §1.

Part 2 — Thorough Study Guide

1 · Exam Details & Case Studies

  • Passing score is 700/1000, a scaled score — not "70% of questions right." Sections are weighted.
  • Question count is not officially published; recent candidates report ~49–54, in ~100–120 minutes of answering time.
  • Question formats: standard multiple choice, case studies, drag-and-drop / build-list ordering, hot area, dropdown, and yes/no repeated-answer series ("does this solution meet the goal?" — the same scenario asked 3 ways, each independent).
  • Renewal: the cert expires after 1 year but renews free via an unproctored online assessment on Microsoft Learn (available 6 months before expiry).

Case-study technique (heavily repeated by passers)

  • A case study is a scenario with tabs: Overview, Business Requirements, Technical Requirements, Existing Environment, then ~6–8 questions.
  • Read the question first, then scan the tabs hunting for constraint keywords — RTO, RPO, cost, region/data-sovereignty, compliance, latency, availability %. Those constraints usually determine the single "best" answer among several that would technically work.
  • Watch out: in some formats you cannot return to a case study once you finish its block and move on — don't rush past it.
  • Use the official exam sandbox (aka.ms/examdemo) beforehand so the drag-and-drop/hot-area UI isn't new on the day.
Mindset trap AZ-305 is not AZ-104. It tests design decisions ("which service, which tier, which topology"), not portal/CLI procedures. Several answers will "work"; the requirements pick the best one.

2 · Well-Architected & Cloud Adoption Frameworks

Every AZ-305 answer is graded against Microsoft's frameworks. Know them as the "why" behind the right choice.

FrameworkPurpose
Well-Architected Framework (WAF)Five pillars for good workloads: Reliability, Security, Cost Optimization, Operational Excellence, Performance Efficiency.
Cloud Adoption Framework (CAF)End-to-end cloud journey: Strategy → Plan → Ready (landing zones) → Adopt (Migrate/Innovate) → Govern → Manage → Secure. Migration questions map here.

Migration "R" strategies: Rehost (lift-and-shift), Refactor, Rearchitect, Rebuild (and Replace). Assess with Azure Migrate.

3 · Identity, Governance & Monitoring 25–30%

3.1 · Identity — Microsoft Entra ID

  • Microsoft Entra ID (formerly Azure AD) is Azure's cloud identity provider. Tiers: Free / P1 / P2.
  • P1 adds Conditional Access, dynamic groups, self-service password reset writeback, and hybrid identity features. P2 adds Identity Protection, Privileged Identity Management (PIM), Access Reviews, and entitlement management.
  • Hybrid identity sync (Entra Connect): Password Hash Sync (simplest), Pass-through Authentication, or Federation (AD FS).
  • Authorization: use RBAC to grant least-privilege access to Azure resources; use Conditional Access for context-aware sign-in (MFA, device/location).
  • B2B (guest collaboration) vs B2C (customer-facing identity).

3.2 · Governance

  • Hierarchy (top→bottom): Management Groups → Subscriptions → Resource Groups → Resources. Apply policy/RBAC at the right scope; children inherit.
  • Azure Policy — enforce/audit rules (allowed regions/SKUs, required tags, enforce encryption). Initiatives group policies. Effects: Deny, Audit, Append, DeployIfNotExists.
  • RBAC vs Policy: RBAC = who can do what; Policy = what resources/config are allowed. Different questions, don't conflate.
  • Resource locks (CanNotDelete / ReadOnly) and tags (cost/organization). Azure Blueprints package policies/RBAC/templates for repeatable environments.
  • Azure Key Vault — store secrets, keys, certificates (Standard vs Premium/HSM-backed). Use managed identities so apps fetch secrets without credentials.

3.3 · Monitoring

  • Azure Monitor is the umbrella. Log Analytics workspace stores logs; query with KQL. Application Insights for app telemetry/APM.
  • Diagnostic settings route resource logs/metrics to Log Analytics, a Storage Account (archive), or Event Hubs (stream to SIEM).
  • Alerts + Action Groups for notification/automation. Microsoft Sentinel = cloud SIEM/SOAR built on Log Analytics.

4 · Data Storage Solutions 20–25%

4.1 · Storage accounts

TypeUse for
BlobObject/unstructured (images, logs, backups); data lake via ADLS Gen2 (hierarchical namespace).
Azure FilesManaged SMB/NFS file share; lift-and-shift shared drives / cloud file server (+ Azure File Sync for hybrid).
Managed DisksBlock storage attached to a single VM (OS/data disks).
Queue / TableSimple message store / NoSQL key-value.
  • Blob access tiers: Hot → Cool → Cold → Archive (Archive = cheapest storage, but rehydration latency + retrieval cost). Lifecycle management auto-tiers.
  • Redundancy (cheapest→most resilient): LRS (1 datacenter) → ZRS (across zones) → GRS (paired region, async) → GZRS (zones + region). RA-GRS/RA-GZRS add read access to the secondary.

4.2 · Relational databases

OptionBest for
Azure SQL DatabaseNew cloud apps; fully PaaS (single DB or elastic pool); no OS/instance access; most managed.
SQL Managed InstanceLift-and-shift — near-100% SQL Server compatibility (SQL Agent, cross-DB queries, CLR); VNet support.
SQL Server on a VM (IaaS)Full OS + instance control; unsupported/legacy features; you patch & manage.
Azure DB for MySQL / PostgreSQLManaged open-source engines.
  • Purchasing: DTU vs vCore. vCore tiers: General Purpose, Business Critical (local SSD + built-in HA replica, lowest latency), Hyperscale (up to 100 TB, fast scale/restore). Serverless auto-pause for intermittent load.
  • HA/scale: active geo-replication, auto-failover groups, read replicas.

4.3 · Cosmos DB & NoSQL

  • Azure Cosmos DB — globally distributed, multi-model, single-digit-ms latency, elastic. APIs: NoSQL (Core), MongoDB, Cassandra, Gremlin (graph), Table. Pick the API to match an existing app/driver.
  • 5 consistency levels (strongest→weakest): Strong → Bounded Staleness → Session (default) → Consistent Prefix → Eventual. Stronger = more latency/cost; weaker = faster.
  • Multi-region writes + global distribution for low-latency global apps.

5 · Business Continuity 15–20%

NeedService
Backup / point-in-time restore (data protection, RPO)Azure Backup (Recovery Services vault) — VMs, files, SQL, etc.
Disaster recovery / regional failover (RTO, near-zero data loss)Azure Site Recovery (ASR) — replicate + orchestrated failover.
  • RTO = how fast you must recover; RPO = how much data loss is acceptable. Case-study answers hinge on these words.
  • HA building blocks: Availability Sets (fault + update domains, single datacenter, 99.95%) vs Availability Zones (physically separate datacenters in a region, 99.99%) vs Region pairs (cross-region DR). VM Scale Sets for autoscaling identical VMs.
  • Data HA: zone-redundant storage (ZRS/GZRS), SQL geo-replication/failover groups, Cosmos multi-region.
Backup vs Site Recovery Backup = keep restorable copies (protects against corruption/deletion; RPO). Site Recovery = replicate a running workload and fail it over to another region fast (RTO). They solve different problems and are often used together.

6 · Infrastructure Solutions 30–35% — biggest

6.1 · Compute & app hosting

6.2 · Networking & load balancing

Memorize the Layer × Scope 2×2 — the most-tested networking decision:

RegionalGlobal
Layer 4 (TCP/UDP)Azure Load BalancerTraffic Manager (DNS-based routing)
Layer 7 (HTTP/S)Application Gateway (URL routing, WAF, SSL offload)Azure Front Door (global HTTP LB + CDN + WAF)
  • Hybrid connectivity: VPN Gateway (encrypted over internet; cheaper; site-to-site / point-to-site) vs ExpressRoute (private dedicated circuit, higher bandwidth/SLA, predictable latency, bypasses public internet). Combine ExpressRoute + VPN for failover.
  • VNet segmentation, NSGs (network security groups), Azure Firewall, Private Endpoints/Private Link, VNet peering + hub-spoke.

6.3 · Messaging & application architecture

ServiceUse for
Service BusEnterprise messaging — queues/topics, ordering (FIFO), transactions, dead-letter.
Event GridReactive routing of discrete events (e.g. blob created) to handlers.
Event HubsHigh-throughput streaming/telemetry ingestion (big data, IoT).
Storage QueuesSimple, large-volume queueing.

Caching: Azure Cache for Redis. API integration: API Management. Config: App Configuration + Key Vault.

6.4 · Migration

  • Azure Migrate — central hub to assess & migrate servers, databases, and apps.
  • Database Migration Service (DMS) for databases; Data Box for offline bulk data transfer.
  • Frame with the CAF and the "R" strategies (rehost/refactor/rearchitect/rebuild).

9 · Study Plan (Retention-Based, ≈3–6 weeks after AZ-104)

AZ-305 rewards understanding trade-offs, not memorizing facts — so lean hard on active recall and elaboration. Reproducing decision trees from memory and explaining why a service fits a constraint is exactly what the case studies test.

PrincipleHow to apply it to AZ-305
Active recallAfter each domain, redraw its decision tree (e.g. SQL DB vs MI vs VM) from a blank page; self-quiz on tiers & SKUs.
Spaced repetitionFlashcard the "which service" pairs and consistency levels/redundancy tiers; review daily at growing intervals.
Elaboration (Feynman)Explain out loud why ExpressRoute beats VPN here, or Backup vs ASR there. Tie every choice to a WAF pillar.
InterleavingMix domains in practice; case studies deliberately blend identity + storage + network + continuity.
Concrete practiceDo case-study-style questions early; the format is the hard part as much as the content.
PhaseFocusActions
Week 0FoundationConfirm AZ-104-level knowledge (VNets, storage, identity, VMs). Fill gaps before starting AZ-305 design content.
Week 1–2Course + framingWork a structured course (Scott Duffy / MS Learn). Watch Savill's Study Cram once to connect the dots. Build decision trees as you go.
Week 3Depth on biggest domainsDeep-dive Infrastructure (30–35%) & Identity/Gov (25–30%). Teach each decision out loud; read Architecture Center patterns.
Week 4Practice + case studiesMeasureUp / Tutorials Dojo in review mode; practice the case-study "read question first, hunt constraints" technique. Re-study weak areas.
Week 5Consistency + taperFresh sets to consistent 80%+. Re-watch Savill's cram 24–48h before. Do the exam sandbox. Sleep well.

Reported times: ~1 week (experienced, MS Learn + Savill) to ~1 month at a few hrs/day after AZ-104; ~100 total hours if newer.

Readiness bar You're ready when you can, from memory, pick the right service for a given constraint set and justify it against a WAF pillar — and you're consistently 80%+ on fresh practice sets.

10 · Tips & Common Traps

  • It's a design exam. Multiple answers "work" — the constraints (RTO/RPO, cost, region/compliance, latency, HA%) pick the best one. Hunt those keywords.
  • Case studies: read the question first, then the tabs. You may not be able to return to a case-study block — don't rush it.
  • Backup vs Site Recovery — restore copies (RPO) vs region failover (RTO).
  • SQL DB vs Managed Instance vs SQL-on-VM — PaaS-new vs lift-and-shift-compat vs full-control.
  • LB vs App Gateway vs Front Door vs Traffic Manager — L4/L7 × regional/global.
  • VPN vs ExpressRoute — internet-encrypted vs private-dedicated.
  • Entra ID P1 vs P2 — P2 adds PIM, Identity Protection, Access Reviews.
  • Availability Set (99.95%, one DC) vs Zone (99.99%, separate DCs).
  • Cosmos consistency: Strong→Bounded→Session(default)→Consistent Prefix→Eventual.
  • Storage redundancy: LRS < ZRS < GRS < GZRS.
  • Know tiers/SKUs — questions hinge on feature differences between tiers.
Ignore Exam dumps (ExamTopics "discussions", DumpsGate, etc.) — often outdated/wrong, violate the NDA, and can void your cert. Legit practice tests (MeasureUp, Tutorials Dojo, Whizlabs) teach reasoning and are fine; dumps just leak stolen answers.
