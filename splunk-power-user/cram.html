Transforming Cmds (5%)

  • Transforming: chart timechart stats top rare contingency. NOT: table, fields, eval.
  • chart <fn> over A by B (1 over + 1 by; over=x-axis).
  • timechart x-axis always _time; span=; only ONE by.
  • top=most common (+count+percent, limit=10); rare=least.
  • stats fns: count sum avg min max values(uniq mv) list(all) dc first last perc95 stdev.

Filter/Format (10%)

  • eval = make/overwrite a field (if, case, coalesce, round, tostring, strftime). Does NOT filter.
  • search = match keywords/values (can repeat mid-pipe).
  • where = boolean eval expr; only one that compares field-to-field + eval fns (like, cidrmatch, isnull).
  • fillnull default replace = 0; fillnull value="N/A".
"A > B (two fields)" → where, not search.

Correlating (15% ★)

transaction

  • Groups events → 1 mv event. Opts: maxspan maxpause startswith endswith.
  • Auto fields: duration, eventcount.
  • Expensive, doesn't scale.

stats

  • stats … by id = fast, scalable. Default choice.
Use transaction ONLY for: keep raw text · start/end conditions · time-bound group. Else → stats.

Fields (10%)

  • Field Extractor (FX): Regex (unstructured, highlight samples) vs Delimiter (structured, comma/tab/pipe).
  • Inline: rex field=_raw "(?<u>\w+)"; erex = example-based.
  • Search-time (schema-on-read).

Data Models + Pivot (10%)

  • Data model = hierarchical semantic map; datasets/objects of type event / search / transaction; attributes (auto, eval, lookup, regex, geo). Children inherit constraints.
  • Pivot = point-click UI on top of a data model (no SPL).
  • Can't use Pivot without a data model.
  • Accelerate → query with tstats.

CIM (10%)

  • Splunk_SA_CIM add-on = standard field names + tags + models (Auth, Network, Web, Malware).
  • Purpose: normalize at search time so apps (ES) work across sources.
  • Make compliant: map fields → CIM names + apply CIM tags.
